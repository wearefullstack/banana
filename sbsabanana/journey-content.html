Nifi ingestion

Apache Nifi is a tool used to automatically ingest data from the data source. Nifi uses processors and flows that have a custom configuration. The nifi processors ingest data from Lithium Social Web’s API, Instagram’s API, Facebook’s API, Twitter’s API and Linkedin’s API. The use of the other APIs was done to enrich the Lithium data further.

Nifi is available at: http://10.143.224.240:9090/nifi

A screenshot of social ingestion processor groups on Nifi. 

Lithium

Lithium Social Web provides data from Facebook, Instagram, Twitter and internal Lithium data which was posted to Standard Bank. The method of ingestion starts with the ingestion of author data. An author is a user on any platform that posts to Standard Bank about queries, concerns or events. Data collected include  information about common trends with questions, feedback or concerns about Standard Bank from the listed social media platform. As this post comes in, a conversation is started. The ingestion then uses the author id to get the conversations of each author per platform. The data is then merged and stored in Hadoop’s filing system.

Instagram

Instagram is a social media platform focused around pictures and media. Instagram is used specifically for events and can be used for heat mapping based on location and tags. Tags index various searches and location is based on activity in a specific region. The data collected includes data from Standard Bank events, popularity of competing banks as well the most popular tags based on area which could aid in marketing. The ingestion of Instagram data is separated into the different processors as each section returns different data. The different processors are for the user information, follow relations, location data, tag data and recent media. Within each section, the process starts with an API call and the data is then stored in Hadoop’s filing system once the data is merged and modified.

Twitter

Twitter is a social media platform focused around tweets and media. Twitter is used specifically for events,quick share of information and live tweeting. This can be used for heat mapping based on location and tags. Tags index various searches as well as  show popularity and location is based on activity in a specific region and events.The data collected included questions, queries and feedback for Standard Bank’s service, events and other business business processes. The ingestion of Twitter data starts with an API call to ingest tweets and the data is then stored in Hadoop’s filing system once the data is merged and modified.

Facebook

Facebook is a social media platform focused around events, information sharing and media. Facebook is used specifically for easy access to information via groups or pages and for user interactivity history. This can be used for data collection with data on opinions and general information. Data collected include tagging of users, information from the Standard Bank page;either posted by Standard Bank or from the customer base, and comments per post about Standard Bank. The ingestion of Facebook data starts with an API call to ingest the data and is then stored in Hadoop’s filing system once the data is merged and modified.

Linkedin

Linkedin is a social media platform focused around professional networking. Linkedin is used specifically for networking, recruiting and advertising personal or company information.The data retrieved includes data about Standard Bank as a company, company updates and about the people in their network. The ingestion of Linkedin data starts with an API call to ingest tweets and the data is then stored in Hadoop’s filing system once the data is merged and modified.
